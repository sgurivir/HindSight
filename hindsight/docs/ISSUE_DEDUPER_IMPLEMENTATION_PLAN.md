# Issue Deduper Implementation Plan

## Overview

The `issue_deduper` is a new deduplication module that removes duplicate issues from the list of issues generated by analyzers (`code_analyzer`, `git_simple_diff_analyzer`, or `trace_analyzer`) **before** HTML report generation. Unlike `issue_tracking_deduper` which deduplicates against external issue tracking systems (like Radar), this deduper works entirely within the analysis pipeline to ensure the generated HTML reports contain only unique issues.

## Goals

1. **Eliminate duplicate issues** in HTML reports generated by analyzers
2. **Reuse existing vector DB infrastructure** from `issue_tracking_deduper`
3. **Create common utilities** that can be shared between both dedupers
4. **Integrate seamlessly** with existing analyzer pipelines

## Key Differences from `issue_tracking_deduper`

| Feature | `issue_tracking_deduper` | `issue_deduper` |
|---------|-------------------------|-----------------|
| Purpose | Dedupe against external issue tracker (Radar) | Dedupe within analysis results |
| Data Source | Downloads issues from Radar API | Uses issues from analyzer output |
| Integration Point | Post-HTML generation (annotates existing reports) | Pre-HTML generation (filters issue list) |
| Output | Annotated HTML with duplicate warnings | Filtered issue list (duplicates removed) |
| Vector DB | Persistent (stores Radar issues) | Session-based (stores current analysis issues) |

```python
@dataclass
class AnalyzerIssue:
    """
    Represents an issue from any analyzer (code, trace, diff).
    
    This is the common format used by issue_deduper.
    """
    id: str                           # Unique identifier (generated)
    title: str                        # Issue title/summary
    description: str                  # Detailed description
    file_path: Optional[str] = None   # Source file path
    function_name: Optional[str] = None  # Function name
    severity: Optional[str] = None    # Severity level
    category: Optional[str] = None    # Issue category
    line_number: Optional[int] = None # Line number
    evidence: Optional[str] = None    # Supporting evidence
    callstack: Optional[List[str]] = None  # For trace analyzer
    
    # Original data preserved for output
    raw_data: Optional[Dict[str, Any]] = None
    
    def to_embedding_text(self) -> str:
        """Generate text for embedding."""
        parts = [self.title, self.description]
        if self.file_path:
            parts.append(f"File: {self.file_path}")
        if self.function_name:
            parts.append(f"Function: {self.function_name}")
        if self.category:
            parts.append(f"Category: {self.category}")
        return " ".join(parts)
    
    def compute_content_hash(self) -> str:
        """Compute hash for exact duplicate detection."""
        content = f"{self.title}|{self.description}|{self.file_path}|{self.function_name}"
        return hashlib.sha256(content.encode()).hexdigest()[:16]
    
    @classmethod
    def from_analyzer_result(cls, result: Dict[str, Any]) -> 'AnalyzerIssue':
        """Create from analyzer output format."""
        pass
    
    def to_dict(self) -> Dict[str, Any]:
        """Convert back to dictionary format."""
        return self.raw_data if self.raw_data else asdict(self)
```

```python
def cosine_distance_to_similarity(distance: float) -> float:
    """Convert ChromaDB cosine distance to similarity score (0-1)."""
    return 1 - (distance / 2)

def get_similarity_threshold(match_type: str = "semantic") -> float:
    """Get appropriate threshold for match type."""
    thresholds = {
        "exact": 0.99,      # Near-identical
        "semantic": 0.85,   # Semantically similar
        "related": 0.70     # Possibly related
    }
    return thresholds.get(match_type, 0.85)

def is_duplicate(similarity: float, threshold: float = 0.85) -> bool:
    """Determine if similarity score indicates a duplicate."""
    return similarity >= threshold
```

## Vector Database Location

The vector database for `issue_deduper` is stored **within the repository's artifacts directory**, not in a global location. This ensures:

1. **Isolation**: Each repository has its own deduplication database
2. **Cleanup**: Database is easily removed with other artifacts
3. **Consistency**: Follows the same directory structure as other analyzer outputs

### Database Lifecycle

**IMPORTANT**: The vector database is **wiped at the start of each deduplication run**. This ensures:

1. **Fresh state**: No stale data from previous runs
2. **Consistency**: All issues in current analysis are compared against each other
3. **No accumulation**: Database doesn't grow unbounded across runs

```python
class IssueDeduper:
    def __init__(self, artifacts_dir: str, ...):
        """
        Initialize the deduper with repository-specific vector DB.
        
        Args:
            artifacts_dir: Path to repository artifacts (e.g., ~/llm_artifacts/corelocation/)
        """
        self.db_path = Path(artifacts_dir) / VECTOR_DB_SUBDIR
        self._wipe_existing_db()  # Always start fresh
        
    def _wipe_existing_db(self) -> None:
        """Wipe existing vector database before starting ingestion."""
        if self.db_path.exists():
            import shutil
            shutil.rmtree(self.db_path)
            logger.info(f"Wiped existing vector DB at: {self.db_path}")
        self.db_path.mkdir(parents=True, exist_ok=True)
```

##### `deduper.py`
Main deduplication orchestrator:

```python
import shutil
from pathlib import Path

class IssueDeduper:
    """
    Main class for deduplicating analyzer issues.
    
    This class orchestrates the deduplication process:
    1. Wipes existing vector DB to ensure fresh state
    2. Ingests issues into a repository-specific vector store
    3. Detects duplicates using semantic similarity
    4. Returns filtered list with duplicates removed
    
    The vector DB is stored in the repository's artifacts directory:
    ~/llm_artifacts/<repo_name>/issue_deduper_db/
    
    Usage:
        deduper = IssueDeduper(
            artifacts_dir="~/llm_artifacts/corelocation",
            threshold=0.85
        )
        unique_issues = deduper.dedupe(issues)
    """
    
    def __init__(
        self,
        artifacts_dir: str,
        threshold: float = DEFAULT_SIMILARITY_THRESHOLD,
        batch_size: int = DEFAULT_BATCH_SIZE
    ):
        """
        Initialize the deduper with repository-specific vector DB.
        
        Args:
            artifacts_dir: Path to repository artifacts directory
                          (e.g., ~/llm_artifacts/corelocation/)
            threshold: Similarity threshold for duplicate detection (0.0-1.0)
            batch_size: Batch size for embedding generation
        """
        self.artifacts_dir = Path(artifacts_dir).expanduser()
        self.db_path = self.artifacts_dir / VECTOR_DB_SUBDIR
        self.threshold = threshold
        self.batch_size = batch_size
        
        # Wipe existing DB before starting (ensures fresh state)
        self._wipe_existing_db()
        
        # Initialize components with the repository-specific DB path
        self.ingester = IssueIngester(
            db_path=self.db_path,
            batch_size=batch_size
        )
        self.detector = DuplicateDetector(
            threshold=threshold,
            vector_store=self.ingester.vector_store
        )
        self._stats = {
            'total_input': 0,
            'duplicates_removed': 0,
            'unique_output': 0,
            'db_path': str(self.db_path)
        }
    
    def _wipe_existing_db(self) -> None:
        """
        Wipe existing vector database before starting ingestion.
        
        This ensures each deduplication run starts with a fresh state,
        preventing stale data from previous runs from affecting results.
        """
        if self.db_path.exists():
            shutil.rmtree(self.db_path)
            logger.info(f"Wiped existing vector DB at: {self.db_path}")
        self.db_path.mkdir(parents=True, exist_ok=True)
        logger.info(f"Created fresh vector DB directory at: {self.db_path}")
    
    def dedupe(self, issues: List[Dict[str, Any]]) -> List[Dict[str, Any]]:
        """
        Deduplicate a list of issues.
        
        Args:
            issues: List of issue dictionaries from analyzer
        
        Returns:
            List of unique issues (duplicates removed)
        """
        if not issues:
            return []
        
        self._stats['total_input'] = len(issues)
        logger.info(f"Starting deduplication of {len(issues)} issues")
        logger.info(f"Vector DB location: {self.db_path}")
        
        # Step 1: Convert to AnalyzerIssue objects
        analyzer_issues = [AnalyzerIssue.from_analyzer_result(i) for i in issues]
        
        # Step 2: Ingest into vector store
        ingested, skipped = self.ingester.ingest(analyzer_issues)
        logger.info(f"Ingested {ingested} issues into vector DB")
        
        # Step 3: Detect duplicates
        duplicates = self.detector.find_duplicates(analyzer_issues)
        logger.info(f"Detected {len(duplicates)} duplicate issues")
        
        # Step 4: Filter out duplicates (keep first occurrence)
        duplicate_ids = {d.duplicate_id for d in duplicates}
        unique_issues = [
            issue.to_dict()
            for issue in analyzer_issues
            if issue.id not in duplicate_ids
        ]
        
        self._stats['duplicates_removed'] = len(duplicate_ids)
        self._stats['unique_output'] = len(unique_issues)
        
        logger.info(
            f"Deduplication complete: {self._stats['total_input']} -> "
            f"{self._stats['unique_output']} issues "
            f"({self._stats['duplicates_removed']} duplicates removed)"
        )
        
        return unique_issues
    
    def get_stats(self) -> Dict[str, int]:
        """Get deduplication statistics."""
        return self._stats.copy()
    
    def get_duplicate_report(self) -> List[DuplicateMatch]:
        """Get detailed report of detected duplicates."""
        return self.detector.get_matches()
    
    def cleanup(self) -> None:
        """
        Clean up resources.
        
        Note: The vector DB is NOT deleted here - it remains in the artifacts
        directory for debugging/inspection. It will be wiped on the next run.
        """
        if hasattr(self, 'ingester'):
            self.ingester.close()
```

```python
from pathlib import Path

class IssueIngester:
    """
    Ingests analyzer issues into a repository-specific vector store.
    
    The vector store is persisted to the repository's artifacts directory,
    allowing for inspection and debugging after analysis completes.
    """
    
    def __init__(
        self,
        db_path: Path,
        batch_size: int = DEFAULT_BATCH_SIZE
    ):
        """
        Initialize the ingester with repository-specific vector store.
        
        Args:
            db_path: Path to the vector database directory
                    (e.g., ~/llm_artifacts/corelocation/issue_deduper_db/)
            batch_size: Batch size for embedding generation
        """
        self.db_path = Path(db_path)
        self.vector_store = VectorStore(
            db_path=self.db_path,
            collection_name=COLLECTION_NAME,
            ephemeral=False  # Persistent storage in artifacts directory
        )
        self.embedding_generator = EmbeddingGenerator.get_instance()
        self.batch_size = batch_size
        logger.info(f"IssueIngester initialized with DB at: {self.db_path}")
    
    def ingest(self, issues: List[AnalyzerIssue]) -> Tuple[int, int]:
        """
        Ingest issues into the vector store.
        
        Args:
            issues: List of AnalyzerIssue objects
        
        Returns:
            Tuple of (ingested_count, skipped_count)
        """
        if not issues:
            return 0, 0
        
        logger.info(f"Ingesting {len(issues)} issues into vector store")
        
        # Generate embeddings in batch
        texts = [issue.to_embedding_text() for issue in issues]
        embeddings = self.embedding_generator.generate_batch(texts, self.batch_size)
        
        # Prepare documents for batch insertion
        documents = []
        for issue, embedding in zip(issues, embeddings):
            documents.append({
                'id': issue.id,
                'text': issue.to_embedding_text(),
                'embedding': embedding,
                'metadata': {
                    'title': issue.title,
                    'file_path': issue.file_path or '',
                    'function_name': issue.function_name or '',
                    'content_hash': issue.compute_content_hash()
                }
            })
        
        ingested, skipped = self.vector_store.add_documents_batch(documents, embeddings)
        logger.info(f"Ingestion complete: {ingested} added, {skipped} skipped")
        return ingested, skipped
    
    def close(self) -> None:
        """Close the vector store connection."""
        if self.vector_store:
            self.vector_store.close()
            logger.debug("Vector store connection closed")
```

##### `duplicate_detector.py`
Detects duplicates using vector similarity:

```python
class DuplicateDetector:
    """
    Detects duplicate issues using semantic similarity.
    
    Two-stage detection:
    1. Exact match: Content hash comparison
    2. Semantic match: Vector similarity above threshold
    """
    
    def __init__(
        self,
        threshold: float = DEFAULT_SIMILARITY_THRESHOLD,
        vector_store: Optional[VectorStore] = None
    ):
        """Initialize the detector."""
        self.threshold = threshold
        self.vector_store = vector_store
        self.embedding_generator = EmbeddingGenerator.get_instance()
        self._matches: List[DuplicateMatch] = []
    
    def set_vector_store(self, vector_store: VectorStore) -> None:
        """Set the vector store to use for queries."""
        self.vector_store = vector_store
    
    def find_duplicates(self, issues: List[AnalyzerIssue]) -> List[DuplicateMatch]:
        """
        Find all duplicates in the issue list.
        
        Uses a greedy approach: for each issue, check if any previously
        seen issue is a duplicate. The first occurrence is kept.
        
        Args:
            issues: List of AnalyzerIssue objects (already ingested)
        
        Returns:
            List of DuplicateMatch objects
        """
        self._matches = []
        seen_hashes: Dict[str, str] = {}  # hash -> issue_id
        seen_ids: Set[str] = set()
        
        for issue in issues:
            # Stage 1: Exact match check
            content_hash = issue.compute_content_hash()
            if content_hash in seen_hashes:
                self._matches.append(DuplicateMatch(
                    original_id=seen_hashes[content_hash],
                    duplicate_id=issue.id,
                    similarity_score=1.0,
                    match_type="exact"
                ))
                continue
            
            # Stage 2: Semantic similarity check
            if self.vector_store and seen_ids:
                query_embedding = self.embedding_generator.generate(
                    issue.to_embedding_text()
                )
                
                # Query excluding the current issue
                results = self.vector_store.query(
                    query_embedding=query_embedding,
                    n_results=1,
                    exclude_ids=[issue.id]
                )
                
                if results:
                    doc_id, metadata, distance = results[0]
                    similarity = cosine_distance_to_similarity(distance)
                    
                    if similarity >= self.threshold and doc_id in seen_ids:
                        self._matches.append(DuplicateMatch(
                            original_id=doc_id,
                            duplicate_id=issue.id,
                            similarity_score=similarity,
                            match_type="semantic"
                        ))
                        continue
            
            # Not a duplicate - add to seen
            seen_hashes[content_hash] = issue.id
            seen_ids.add(issue.id)
        
        return self._matches
    
    def get_matches(self) -> List[DuplicateMatch]:
        """Get all detected duplicate matches."""
        return self._matches.copy()
```

```python
# tests/test_issue_deduper.py

def test_exact_duplicate_detection():
    """Test that exact duplicates are detected."""
    issues = [
        {"issue": "Memory leak in function A", "file": "a.py", "function": "func_a"},
        {"issue": "Memory leak in function A", "file": "a.py", "function": "func_a"},  # Exact duplicate
    ]
    deduper = IssueDeduper()
```

## Performance Considerations

1. **Ephemeral Vector Store**: Using in-memory ChromaDB avoids disk I/O overhead
2. **Batch Embedding Generation**: Process embeddings in batches for efficiency
3. **Early Exit for Exact Matches**: Check content hash before computing embeddings
4. **Lazy Loading**: Load embedding model only when needed

## Summary

The `issue_deduper` module provides a clean, efficient solution for removing duplicate issues from analyzer output before HTML report generation. By extracting common functionality into a shared module, we maintain code reuse and consistency between the two dedupers while keeping their distinct purposes clear.

### Key Benefits

1. **Cleaner Reports**: HTML reports contain only unique issues
2. **Code Reuse**: Common vector DB and embedding functionality shared
3. **Minimal Integration**: Simple integration point in existing analyzers
4. **Configurable**: Threshold and batch size can be tuned per use case
5. **Efficient**: In-memory processing with batch operations
